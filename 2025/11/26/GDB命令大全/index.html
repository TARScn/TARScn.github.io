GDB命令大全

GDB 命令大全（超全版）

一、启动与退出

  • gdb <program>
    启动 GDB 并加载可执行文件。

  • gdb <program> <core>
    调试 core dump 文件。

  • gdb -tui <program>
    使用 TUI 模式（分屏显示代码和调试信息）。

  • file <program>
    在 GDB 内加载可执行文件。

  • quit / q
    退出 GDB。

二、运行与进程控制

  • run / r
    运行程序（可附带参数）。

  • run arg1 arg2
    运行程序并传参。

  • set args arg1 arg2
    设置程序运行时的默认参数。

  • show args
    查看当前参数设置。

  • attach <pid>
    附加调试正在运行的进程。

  • detach
    从进程分离，不终止程序。

  • kill
    终止正在调试的程序。

三、断点（Breakpoints）

1. 设置断点

  • break <line> / b <line>
    在指定行设置断点。

  • break <function>
    在函数入口处设置断点。

  • break <file:line>
    指定文件中的某一行断点。

  • break *<address>
    在指定内存地址设置断点。

  • tbreak <location>
    设置一次性断点（触发一次后自动删除）。

2. 条件断点

  • condition <num> <expr>
    给断点加条件。

3. 管理断点

  • info breakpoints / i b
    查看所有断点。

  • delete <num> / d <num>
    删除断点。

  • disable <num>
    禁用断点。

  • enable <num>
    启用断点。

四、程序执行控制

  • continue / c
    恢复执行直到下一个断点。

  • next / n
    单步执行（不进入函数）。

  • step / s
    单步执行（会进入函数）。

  • finish
    执行直到当前函数返回。

  • until
    运行直到当前循环或函数的下一行。

  • jump <line/address>
    强制跳转到某行或某地址。

五、查看源码

  • list / l
    查看源码（每次显示 10 行）。

  • list <line>
    查看特定行附近的代码。

  • list <function>
    显示某函数代码。

  • 重复 list
    显示下一段代码。

六、查看栈与调用信息

  • backtrace / bt
    打印调用栈。

  • backtrace full
    打印调用栈和每一帧的局部变量。

  • frame <num> / f <num>
    切换到指定栈帧。

  • up
    切换到上层栈帧。

  • down
    切换到下层栈帧。

  • info args
    显示当前函数的参数。

  • info locals
    显示当前函数的局部变量。

七、打印变量 / 内存 / 类型

1. 打印变量

  • print <expr> / p <expr>
    打印变量或表达式值。

  • p /x <expr>
    以十六进制打印。

  • display <expr>
    每次单步后自动显示表达式。

  • undisplay <num>
    删除自动显示项。

2. 查看内存（x 命令）

格式： x /<n><format><size> <address>

常用格式：

  • x/10xb <addr> — 查看 10 个字节（hex）
  • x/4i <addr> — 反汇编 4 条指令
  • x/20xw <addr> — 查看 20 个 4 字节字(word)

3. 查看类型

  • ptype <var>
    显示变量类型。

  • whatis <expr>
    显示表达式类型。

八、寄存器和汇编级调试（重要）

1. 汇编级单步

  • stepi / si
    执行一条机器指令。

  • nexti / ni
    跳过函数执行一条机器指令。

2. 查看汇编

  • disassemble
    反汇编当前函数。

  • disassemble <address>
    反汇编指定地址范围。

3. 寄存器

  • info registers
    查看所有寄存器。

  • p $rax
    打印寄存器（如 x86_64 的 rax）。

九、线程调试（多线程程序）

  • info threads
    查看所有线程。

  • thread <id>
    切换到指定线程。

  • thread apply all <cmd>
    对所有线程执行命令。

  • thread apply <id> <cmd>
    只对某线程执行命令。

十、多进程（fork / exec）

  • set follow-fork-mode child|parent
    设置调试 fork 后跟踪谁。

  • set detach-on-fork on|off
    控制是否同时调试两进程。

  • info inferiors
    查看当前所有被调试的进程。

  • inferior <id>
    切换当前进程。

十一、Watchpoint（监视变量）

  • watch <expr>
    当表达式的值发生变化停止。

  • rwatch <expr>
    读表达式时停止。

  • awatch <expr>
    读写任何访问都停止。

  • info watchpoints
    查看所有 watchpoints。

十二、脚本 / 自动化

  • source <file>
    运行一组 GDB 命令（脚本）。

  • .gdbinit
    在当前目录或主目录下自动执行。

十三、其他常用命令

  • help
    显示帮助。

  • help <command>
    查看某命令帮助。

  • set pagination off
    关闭分页显示（打印长输出不需按回车）。

  • set print pretty on
    格式化输出结构体内容。

  • info functions
    显示所有函数列表。

  • info variables
    显示所有全局变量。
